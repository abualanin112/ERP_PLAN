## Module Hygiene Exercises (repo-specific)

تم تصميم هذه التمارين لتطبيق مهارات المعمارية مباشرةً على المستودع الحالي (`notes`, `iam`, `audit`). كل تمرين يجب أن ينتج PR صغير قابل للمراجعة.

- **Refactor flat module (`notes`)** — هدف: تحقق أن `notes` يتبع بنية المسطحة ويصدّر واجهة عامة صغيرة. مخرجات: PR يُظهر تبسيط `index.js` وملفّات محسّنة. الوقت: 0.5 يوم.
- **Flatten `iam` module** — هدف: إزالة التجزئة غير الضرورية (folders-per-file) وترك مجلد واحد مع واجهة تسجيل Routes/Hooks. مخرجات: PR مع إعادة ترتيب الملفات وفحص الاعتمادات. الوقت: 1 يوم.
- **Consolidate anemic layers (example: `audit`)** — هدف: دمج `service` و`repository` البسيطة إذا كانت الأنشطة مجرد استدعاءات Prisma. مخرجات: PR مع اختبارات وحدة تغطي السلوك قبل وبعد الدمج. الوقت: 0.5-1 يوم.
- **Enforce module index policy** — هدف: إزالة `export *` من ملفات `index.js` واستبدالها بتصدير واجهة عامة محدودة. مخرجات: PR وتوثيق قصير في `copilot-erp-architect-instructions.md`. الوقت: 0.5 يوم.

## Validation & Acceptance Criteria

- كل PR يجب أن يحتوي على: وصف قصير للتغيير، قائمة ملفات متأثرة، أسباب التصميم، وفحص تأثير الأداء/التوثيق.
- تجارب الدمج يجب أن تكون مغطاة باختبارات وحدة بسيطة وشرح إعادة التشغيل (how to run tests).

أكمل هذه التمارين لتحويل المعرفة النظرية إلى تحسينات عملية واضحة في المستودع.

## Project-specific Practical Tasks (expert track)

These exercises map the roadmap to the actual PropTech codebase. Each item should be attempted in sequence and produce a small, testable deliverable.

- **Pre-signed upload flow (R2):** deliverable: API spec for presign endpoint + minimal React uploader sample + automated test that uploads a small file to a staging bucket. Time: 1-2 days.
- **Contract Text-Versioning:** deliverable: Prisma model + migration + `createContractVersion()` service and an integration test that verifies snapshot retrieval for a booking. Time: 1-2 days.
- **OCR Worker Pipeline:** deliverable: BullMQ job definition, worker that calls Google Vision (mockable), and parser functions (regex + normalization) with unit tests. Time: 2-3 days.
- **iCal Sync Job:** deliverable: a scheduled worker using `node-ical` that ingests an external feed and emits availability events; include conflict resolution rules using `date-fns`. Time: 1-2 days.
- **Neon Pooler + Backup Automation:** deliverable: `.env` guidelines, `pg_dump` cron script (or job), and verification steps for R2 upload. Time: 0.5-1 day.
- **Future Cache Invalidation Pattern:** deliverable later: `CacheService` skeleton, pub/sub invalidation example and integration test showing cross-instance invalidation. Do this after monitoring proves read-cache need. Time: 1 day.

## Proficiency Checklist (how to validate mastery)

- Write at least one small feature end‑to‑end (API → worker → DB) with tests; add cache later as an optimization exercise after baseline performance is measured.
- Provide a short architecture note (1 page) explaining trade-offs and monitoring plan.
- Demonstrate ability to reason about scaling: connection pooling, partitioning, and cache eviction strategies.

Complete these practical tasks to move from conceptual knowledge to applied expertise in this codebase.
